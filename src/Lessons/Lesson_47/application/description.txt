Приложение для обмена сообщениями.

Модели приложения:

 - Сообщение:
        String отправитель
        String получатель
        String текст_сообщения
        LocalDateTime время_отправки (привязка к временной зоне)
        boolean доставлено
        boolean прочитано

 - Пользователь:
        String имя (nick)
        String номер
        String system_id
        boolean is_online



Репозитории:
Базовый репозиторий всегда выполняет 4 операции:
CRUD - create, read, update, delete (создать, читать, изменять, удалять)

 - ХранилищеПользователей:
        - поиск_по_имени
        - поиск_по_номеру
        - показать_тех_кто_онлайн
        - добавить_пользователя
        - удалить_пользователя
        - изменять_запись


 - ХранилищеСообщений:
        - добавить_сообщение
        - получить_сообщение_по_пользователю
        - получить_сообщения_не_прочитанные
        - получить_сообщения_за_конкретную_дату
        - метод_изменения_сообщений
        - удалить_сообщение

Бизнес логика (сервисы):

    Сервис_регистрации_пользователя:
        - создать пользователя
        - изменить пользователя
        - удалить пользователя

    Сервис_отправки_сообщений:
        - отправить_сообщение (сообщение, пользователь)
        - получить_сообщение (сообщение)
        - прочитать_сообщение (сообщение)
        - удалить_сообщение (сообщение)


Слой представления:
    класс, который позволяет работать с бизнес логикой.
    Единственный класс, к которому имеет доступ пользователь, то есть фронтенд

    Должен иметь доступ к методам из бизнес слоя
